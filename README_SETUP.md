# HR Automation System - Инструкция по запуску

## Требования

Перед запуском убедитесь, что установлены:

1. **PostgreSQL** (версия 12 или выше)
   - Скачать: https://www.postgresql.org/download/
   - При установке запомните пароль для пользователя `postgres`
   - По умолчанию используется пароль: `0206`

2. **.NET 8 SDK**
   - Скачать: https://dotnet.microsoft.com/download/dotnet/8.0
   - Выберите SDK (не Runtime)

## Быстрый запуск

### Вариант 1: Автоматический запуск (рекомендуется)

Просто запустите файл:
```
start.bat
```

Скрипт автоматически:
- ✓ Проверит наличие PostgreSQL и .NET SDK
- ✓ Создаст базу данных `hr_automation` (если не существует)
- ✓ Применит все миграции
- ✓ Соберет приложение
- ✓ Запустит приложение

### Вариант 2: Ручной запуск

Если нужно запустить вручную:

1. Создайте базу данных:
```bash
psql -U postgres -c "CREATE DATABASE hr_automation;"
```

2. Примените миграции:
```bash
cd Automation
dotnet ef database update
```

3. Запустите приложение:
```bash
dotnet run
```

## Учетные записи

После запуска используйте эти учетные данные для входа:

| Роль | Логин | Пароль |
|------|-------|--------|
| Администратор | `admin` | `admin123` |
| HR Менеджер | `hr` | `hr123` |

## Настройка подключения к БД

Если у вас другие параметры PostgreSQL, отредактируйте файл:
- `Automation/appsettings.json` - строка подключения
- `setup-database.ps1` - параметры в начале файла

Пример строки подключения:
```json
{
  "ConnectionStrings": {
    "DefaultConnection": "Host=localhost;Port=5432;Database=hr_automation;Username=postgres;Password=ВАШ_ПАРОЛЬ"
  }
}
```

## Возможные проблемы

### PostgreSQL не запущен
Запустите службу PostgreSQL:
- Windows: Службы → postgresql-x64-XX → Запустить
- Или через pgAdmin

### Порт 5432 занят
Измените порт в `appsettings.json` и `setup-database.ps1`

### Ошибка миграций
Попробуйте удалить БД и создать заново:
```bash
psql -U postgres -c "DROP DATABASE hr_automation;"
psql -U postgres -c "CREATE DATABASE hr_automation;"
cd Automation
dotnet ef database update
```

## Возможности системы

### Администратор (admin)
- ✓ Полный доступ ко всем разделам
- ✓ Управление сотрудниками (включая увольнение)
- ✓ Одобрение/отклонение отпусков
- ✓ Управление зарплатами и табелем
- ✓ Просмотр отчетов

### HR Менеджер (hr)
- ✓ Управление сотрудниками (без увольнения)
- ✓ Добавление заявок на отпуск
- ✓ Управление отделами и должностями
- ✓ Просмотр отчетов
- ✗ Нет доступа к зарплатам и табелю
- ✗ Не может одобрять отпуска

## Фильтрация и сортировка

### Сотрудники
- Поиск по ФИО и email
- Фильтр по отделу и должности
- Сортировка по фамилии, дате приема, возрасту

### Отпуска
- Фильтр по статусу и типу
- Сортировка по дате, длительности, сотруднику

## Техническая поддержка

При возникновении проблем проверьте:
1. Запущен ли PostgreSQL
2. Правильный ли пароль в `appsettings.json`
3. Установлен ли .NET 8 SDK (не Runtime)
4. Доступен ли порт 5432

Для сброса БД:
```bash
psql -U postgres -c "DROP DATABASE hr_automation;"
.\start.bat
```
